## [4.3.0] - 2026-02-26

### ✨ 新功能：代码可维护性和可扩展性质量评分

#### 核心新功能
- ✨ **可维护性评分**: 自动评估代码的可维护性（注释率、函数长度、嵌套深度、命名质量等）
- ✨ **可扩展性评分**: 自动评估代码的可扩展性（接口使用、设计模式、模块化、抽象化程度等）
- ✨ **双维度质量分析**: 综合评估代码质量（可维护性 60% + 可扩展性 40%）
- ✨ **设计模式识别**: 自动检测工厂、单例、策略、观察者、建造者等模式
- ✨ **智能等级评定**: 优秀/良好/中等/较差/差 五级评定

#### 可维护性指标（8项）
- 📝 注释率（10-30% 最佳，含 TODO 和文档注释统计）
- 📏 函数平均长度（建议 < 50 行）
- 📏 最大函数长度（建议 < 100 行）
- 🔀 嵌套深度（建议 < 4 层）
- 🔢 魔法数字检测（硬编码数字）
- 🏷️ 命名质量评估（有意义的命名比例）
- 📝 待办事项统计（TODO/FIXME）
- 📊 代码行数统计

#### 可扩展性指标（9项）
- 🎯 接口使用（TypeScript Interface）
- 🏗️ 抽象类（Abstract Class）
- 📦 类型定义（Type Definitions）
- 🔄 泛型使用（Generics）
- 🧩 模块化程度（Exports/Imports 比例）
- 📐 抽象化程度（接口/抽象类占比）
- 🎨 设计模式应用（5种常见模式）
- 🔓 开闭原则得分（OCP Score）
- 🎭 装饰器使用（Decorators）

#### 设计模式检测
- 🏭 工厂模式（Factory Pattern）
- 🔒 单例模式（Singleton Pattern）
- 🎯 策略模式（Strategy Pattern）
- 👀 观察者模式（Observer Pattern）
- 🏗️ 建造者模式（Builder Pattern）

#### 新增文件
- 📦 `core/maintainability.js`: 可维护性和可扩展性分析核心模块
- 📄 `MAINTAINABILITY_GUIDE.md`: 详细使用指南和最佳实践

#### 功能集成
- ✅ 集成到 `compare.js`（个人比对）
- ✅ 集成到 `branch-multi-compare.js`（多分支多人比对）
- ✅ 报告中新增可维护性和可扩展性对比表
- ✅ JSON/Markdown/HTML 报告包含完整质量数据

#### 报告展示示例

## [4.2.1] - 2026-02-26

### 🐛 Bug修复

#### 修复：多分支多开发者比对commit数量不准确
- 🔧 **问题**: 多分支多开发者比对获取的commit数量比个人比对少很多
- 🔧 **原因**: 原先逐个分支查询只能获取指定分支的commits，遗漏了其他分支的提交
- ✅ **修复**: 改用 `getCommits` 一次性获取所有指定分支的commits（包括这些分支可达的所有commits）
- 📊 **改进**: 新增日志输出，显示"通过分支查询"和"通过完整查询"获取的commits数量对比
- 📝 **详细说明**: 查看 [FIX_COMMIT_COUNT.md](FIX_COMMIT_COUNT.md)

**影响范围**:
- `branch-multi-compare.js` - 修改了获取commits的逻辑
- 现在能更准确地统计开发者的完整贡献，不会遗漏任何commits

**Git命令对比**:
- 修复前: `git log main` + `git log develop`（分别查询后合并）
- 修复后: `git log main develop`（一次性获取所有可达commits）

## [4.2.0] - 2026-02-25

### 🎯 团队自动化打分系统（重大更新）

#### 核心新功能
- ✨ **多维度自动打分**: 基于6大维度自动计算团队成员综合得分
- ✨ **智能等级评定**: S/A/B/C/D 五级评定系统，含性能系数
- ✨ **相对排名**: 在团队内进行相对评分，更公平客观
- ✨ **优劣分析**: 自动识别每个成员的优势领域和改进方向
- ✨ **可视化报告**: 柱状图展示各维度得分，一目了然
- ✨ **个人详细报告**: 🎁 为每个员工生成单独的个人评估报告

#### 新增工具
- 📦 `score-team.js`: 团队成员自动化综合打分工具
- 🔧 新增命令: `eng-score` 或 `npm run score`

#### 个人报告功能（新增）
- 📝 **自动生成**: 为每个员工生成 Markdown 和 HTML 两种格式的个人报告
- 🎯 **详细分析**: 包含6个维度的详细得分、团队对比、加权贡献
- 💪 **优势识别**: 自动识别前3项优势领域，作为核心竞争力
- 🎯 **改进建议**: 针对薄弱环节提供3-4条具体可执行的行动建议
- 📊 **数据对比**: 与团队平均、最高、最低水平的详细对比
- 🚀 **发展规划**: 根据等级（S/A/B/C/D）提供个性化的职业发展建议
- 🎨 **精美设计**: HTML报告采用渐变色设计，视觉效果出色

**个人报告文件**：
- `individual-<员工名>.md` - Markdown格式，便于编辑和打印
- `individual-<员工名>.html` - HTML格式，可直接发送给员工查看

**个人报告内容**：
1. 综合评分：总分、等级、排名、性能系数
2. 各维度详细分析：分数、团队对比、加权贡献
3. 优势分析：前3项优势领域及水平评价
4. 改进方向：3个重点改进维度 + 每维度4条具体建议
5. 原始数据统计：12项关键指标的完整数据
6. 发展建议：基于等级的个性化职业发展规划

#### 评分维度（可自定义）
- 🎯 **工程价值** (25%): 平均分数、总分数 - 整体工程贡献
- 💎 **代码质量** (25%): 质量评分、复杂度控制 - 代码健壮性
- 💼 **工作量** (20%): 提交数、代码行数 - 工作产出
- 🚀 **技术影响力** (15%): 影响分数、文件数 - 技术广度
- 🧠 **代码复杂度** (10%): 平均/最大复杂度 - 处理难题能力
- 🏗️ **AST指标** (5%): 函数/类/接口/Hooks - 架构能力

#### 等级标准
- **S级** (90-100分): 🏆 卓越表现 - 性能系数 1.5x
- **A级** (80-89分): 🥇 优秀表现 - 性能系数 1.2x
- **B级** (70-79分): 🥈 良好表现 - 性能系数 1.0x
- **C级** (60-69分): 🥉 合格表现 - 性能系数 0.8x
- **D级** (<60分): 📊 需改进 - 性能系数 0.6x

#### 使用方法
```bash
# 1. 生成对比报告
node compare.js 2024 "张三" "李四" "王五"
# 或
node branch-multi-compare.js 2024 main develop -- "张三" "李四" "王五"

# 2. 自动打分
node score-team.js ./reports/comparison-report-2024.json

# 使用全局命令
eng-score ./reports/comparison-report-2024.json
```

#### 输出文件
- `*-scoring.json` - 完整的评分数据和详细维度
- `*-scoring.md` - Markdown 格式评分报告
- `*-scoring.html` - 可视化评分报告（强烈推荐）

#### 核心特性
- ✅ **相对评分**: 在团队内相对比较，避免绝对标准的局限性
- 📊 **全面维度**: 6个维度全方位评估，不遗漏任何贡献
- 🎯 **个性化评价**: 针对不同等级生成具体评价和改进建议
- 🔧 **可自定义**: 支持调整各维度权重，适应不同团队需求
- 📈 **可视化**: 直观的柱状图和表格展示

#### 应用场景
- 📋 **绩效评估**: 为绩效考核提供量化依据
- 🏆 **团队排名**: 识别团队中的技术标杆
- 💡 **个人发展**: 明确每个成员的优势和改进方向
- 🎯 **激励计划**: 基于性能系数制定奖励方案
- 📊 **定期回顾**: 季度/年度团队技术复盘

#### 文档更新
- 📖 更新 `README.md`: 添加自动打分工具说明
- 📖 更新 `QUICKSTART.md`: 包含打分工具快速使用指南
